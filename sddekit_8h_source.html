410 /* Structs used to simplify system interface
411  *
412  * Previous docs on system definition callback:
413  *
414  * The coupling term array has two uses. On entry to system callback,
415  * the array contains elements for every delayed term defined using the
416  * nc/vi/vd arguments to sd_sol_init, which are used to define the delayed
417  * coupling structure of the system. On exit, the array should contain
418  * values to be used/stored for delayed coupling. The array vi determines
419  * the mapping from efferent coupling terms to afferent terms.
420  *
421  * \note The partial derivative terms F, G, Cf, Cg should only be set if
422  * the scheme requires them; they will be NULL otherwise. F & G have a dense
423  * nx x nx row-major matrix layout whereas Cf & Cg have the same sparse layout
424  * as provided to the solver.
425  *
426  * \param data user data passed to system for defining e.g. parameters of the system.
427  * \param hist history of system.
428  * \param t current time in solution.
429  * \param i index of system (relevant for composite systems).
430  * \param nx number of state variables of system.
431  * \param x vector of current state variable values.
432  * \param f vector of drift terms per state variable.
433  * \param f vector of diffusion terms per state variable.
434  * \param F partial derivatives of drift terms with respect to x.
435  * \param G partial derivatives of diffusion terms with respect to x.
436  * \param nc number of coupling terms.
437  * \param c vector of coupling terms.
438  * \param Cf partial derivatives of drift terms with respect to c.
439  * \param Cg partial derivatives of diffusion terms with respect to c.
440  * \return 0 if system evaluation succeeded, 1 if error occurred.
441  */

603 /* network
604  * \brief sd_net provides a sd_sys which adapts another sd_sys into a network.
605  *
606  * The solver & scheme sees the network as just another
607  * system to solve, and the network structure is handled here, not in the
608  * solver.
609  *
610  * We consider in general n nodes and m mass models. The i'th node has model
611  * M[i], with Ms[i] state variables and Me[i] efferent variables. ns =
612  * sum(Ms[i],i,1,n) is the total number of state variables, and ne =
613  * sum(Me[i],i,1,n) respectively efferent variables. x in R^ns is the
614  * state of the network, and ce in R^ne the efferent activity.
615  *
616  * Connectivity in the network is a sparse R^ne x R^ne matrix, where
617  * w in R^nnz is the non-zero weights and d in R^nnz
618  * the delays. The non-zero elements are located by Or in N^(n+1) row
619  * offsets and Ic in N^nnz column indices, in so-called CSR format.
620  *
621  * The historian requires Ic in N^nnz indices on ce, and
622  * d in R^nnz delay vector, also passed for initializing a network.
623  *
624  * Computing net afferent cna in R^ne from sys aff ca in R^nnz following
625  * the sparse connectivity.
626  *
627  * scheme provides ca from history, expects ce, we need to map these
628  * sparse needs n, row offsets col indices and data
629  *
630  */
631
